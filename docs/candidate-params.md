# 候选词生成参数说明

Sime 的候选词生成采用两层策略（Two-Layer Strategy），与 fcitx5-pinyin/libime 的行为对齐。本文档详细说明所有影响出词顺序和数量的参数。

## 用户可配置参数

这些参数可在 Fcitx5 设置界面中调整，保存在 `~/.config/fcitx5/conf/sime.conf`。

### PageSize（每页候选数）

- **位置**: `fcitx5/sime.h` Config
- **默认值**: 9
- **作用**: 候选词列表每页显示的数量，对应数字键 1-9 选词。
- **影响**: 仅影响 UI 分页，不影响实际生成的候选总数。

### NBest（候选总数）

- **位置**: `fcitx5/sime.h` Config
- **默认值**: 36
- **作用**: 传给 `DecodeSentence()` 的 `max_top` 参数，控制最终返回的候选词上限。
- **影响**: 直接决定用户能翻页看到的候选词总数。两层候选合并后，超出此数量的部分会被截断。
- **参考**: libime 默认也使用类似的候选总数上限。

## 解码器内部参数

以下参数为编译期常量，定义在 `src/interpret.cc` 的 `DecodeSentence()` 函数中。

### beam（Beam 宽度）

- **计算公式**: `max(max_top * 3, 40)`
- **默认值**: max(36×3, 40) = 108
- **作用**: 每一列（lattice column）保留的最大搜索状态数。Beam 越大，搜索空间越广，能发现更多候选路径。
- **影响**: 是所有候选生成的基础。Beam 过小会导致好的路径被剪枝丢弃。
- **参考**: libime 默认 beamSize=20, frameSize=40。Sime 使用较大的 beam 是因为需要同时支撑两层候选的生成。

### kNBest（Layer 1 全句候选数）

- **值**: 2
- **作用**: Layer 1 从完整匹配路径中取前 N 个最佳整句候选。这些候选覆盖全部输入拼音，排在候选列表最前面。
- **影响**: 控制整句候选的数量。设为 2 表示最多取 2 个不同的整句翻译。
- **排序**: 按语言模型分数排序，分数高的在前。
- **参考**: libime 默认 nbest=2。

### kPerPrefix（Layer 2 每列候选上限）

- **值**: 15（全匹配列不限）
- **作用**: Layer 2 遍历 lattice 的每一列，从每列的 beam 状态中提取词/短语候选。此参数限制每列最多提取多少个状态来生成候选。
- **特殊规则**:
  - **全匹配列**（col == effective_n）: 不设上限，遍历该列所有 beam 状态。因为全匹配候选覆盖了所有输入，质量最高，应该尽量多出。
  - **部分匹配列**: 上限为 15。部分匹配只覆盖输入的前缀部分，靠后的列距离越远惩罚越大，限制数量可以避免低质量候选过多。
- **影响**: 这是影响出词数量的关键参数。过小（如之前的 5）会导致短输入候选太少。
- **参考**: libime 的 wordCandidateLimit=15（单字候选不限）。

### kDistancePenaltyFactor（距离惩罚因子）

- **值**: 3.0
- **计算**: `penalty_per_unit = abs(unknownPenalty) / kDistancePenaltyFactor`
- **作用**: Layer 2 的部分匹配候选会根据未匹配的拼音距离施加惩罚。距离越远（即候选只匹配了输入的前半部分），惩罚越重。
- **影响**: 控制部分匹配候选的排序位置。因子越大，惩罚越轻，部分匹配候选排名越靠前；因子越小，惩罚越重，部分匹配候选排名越靠后。
- **公式**: 对于匹配了前 k 个音节的候选，距离为 `effective_n - k`，惩罚为 `distance × penalty_per_unit`。最终分数 = 原始分数 - 距离惩罚。
- **参考**: libime 使用 `unknownPenalty / 1.8` 作为距离惩罚。Sime 使用 `/3.0` 惩罚更轻，让部分匹配候选更容易出现。

## 两层策略概览

```
输入: "nihao"  →  解析为音节 [ni, hao]  →  构建 lattice

Layer 1 (整句候选, 最多 kNBest=2 个):
  ┌─────────────────────────────────┐
  │ 你好  (score: -23.3, 全匹配)     │  ← 覆盖全部输入，排最前
  │ 呢好  (score: -27.9, 全匹配)     │
  └─────────────────────────────────┘

Layer 2 (词/短语候选):
  col=1 (匹配 "ni", 距离=1):
  ┌─────────────────────────────────┐
  │ 呢  你  拟  尼  泥  ...          │  ← 最多 kPerPrefix=15 个
  │ 每个扣除 1×penalty_per_unit      │
  └─────────────────────────────────┘

  col=2 (匹配 "nihao", 距离=0, 全匹配列):
  ┌─────────────────────────────────┐
  │ 你号  妳好  呢号  拟好  ...       │  ← 不限数量
  │ 无距离惩罚（与 Layer 1 去重）     │
  └─────────────────────────────────┘

合并 → 按分数排序 → 截断到 NBest=36 → 返回
```

## 参数调优建议

| 问题 | 调整方向 |
|------|---------|
| 候选总数太少 | 增大 NBest、增大 kPerPrefix |
| 短输入（1-2个字）候选少 | 增大 kPerPrefix 或取消全匹配列的限制（已实现） |
| 部分匹配候选排名过低 | 增大 kDistancePenaltyFactor（减轻惩罚） |
| 部分匹配候选排名过高，挤掉整句 | 减小 kDistancePenaltyFactor（加重惩罚） |
| 搜索不到某些词 | 增大 beam 宽度 |
| 解码速度慢 | 减小 beam、减小 kPerPrefix |
